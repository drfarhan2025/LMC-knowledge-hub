Day 1 — Monday, 14th April
What are Large Language Models (LLMs)? + Intro to Prompting
Concept: Understand what an LLM is (a system trained to predict text), and why the quality of your input (prompt) directly determines the quality of output.

Key Points:
• What is an LLM? (Not magic, not "thinking" — pattern matching at scale)
• How does Claude/ChatGPT/Gemini work?
• Why prompting matters: Garbage in = Garbage out
• Difference between a vague prompt and a clear one (examples)

Day 2 — Tuesday, 15th April
The Anatomy of an Effective Prompt
Concept: Learn the structure that makes prompts work: Context → Task → Format → Constraints.

Key Points:
• Role-playing: "You are a..." (gives context)
• Clear instruction: What exactly do you want?
• Output format: How should the answer look? (bullet points, paragraph, table, etc.)
• Constraints: Length, tone, language, style
• Real clinic example: Writing a patient discharge instruction prompt

Day 3 — Wednesday, 16th April
Markdown: The Language of Clean Text
Concept: Markdown is a simple way to format text so it looks organized. LLMs understand markdown perfectly—use it in prompts to get better structure.

Key Points:
• What is Markdown? (Plain text with formatting symbols)
• Syntax: Headers (#), Bold (**), Lists (-, •), Code blocks (```), Tables, Links
• Why use it? Clarity. Readability. LLMs respond better to organized input.
• Example: A markdown-formatted clinic SOP vs. messy paragraph

Day 4 — Thursday, 17th April
Prompting in Markdown + Generating Files
Concept: Combine what you learned (good prompting + markdown clarity) to ask LLMs to generate actual files: Word docs, Excel sheets, HTML pages, PDFs, images, even videos.

Key Points:
• How to ask an LLM to generate files (request format, file type, structure)
• File types LLMs can create:
  - Markdown (.md) — documents, SOPs, guides
  - HTML (.html) — web pages, forms, dashboards
  - Word (.docx) — formatted reports, letters, templates
  - Excel (.xlsx) — spreadsheets, budgets, trackers
  - JSON/CSV — data structures, databases
  - Python/Code — scripts that automate tasks
  - Images (via image generation AI) — diagrams, illustrations
• Real clinic example: Generating a patient intake form in HTML using a markdown prompt
